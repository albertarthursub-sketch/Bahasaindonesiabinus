================================================================================
                         QUICK START COMMANDS
================================================================================

AUTOMATED SETUP (Recommended):
  bash setup-secret-manager.sh

MANUAL SETUP:
  # 1. Enable Secret Manager API
  gcloud services enable secretmanager.googleapis.com

  # 2. Create secrets
  echo -n "arthurapp05@gmail.com" | gcloud secrets create TEACHER_EMAIL_USER --data-file=-
  echo -n "your-app-password" | gcloud secrets create TEACHER_EMAIL_PASSWORD --data-file=-

  # 3. Grant access
  PROJECT_ID=$(gcloud config get-value project)
  FUNCTIONS_SA="${PROJECT_ID}@appspot.gserviceaccount.com"
  gcloud secrets add-iam-policy-binding TEACHER_EMAIL_USER --member="serviceAccount:${FUNCTIONS_SA}" --role="roles/secretmanager.secretAccessor"
  gcloud secrets add-iam-policy-binding TEACHER_EMAIL_PASSWORD --member="serviceAccount:${FUNCTIONS_SA}" --role="roles/secretmanager.secretAccessor"

  # 4. Deploy
  cd functions && npm install && npm run build
  firebase deploy --only functions
  firebase deploy --only firestore:rules

  # 5. Verify
  firebase functions:log

================================================================================
                       ARCHITECTURE OVERVIEW
================================================================================

PRODUCTION ARCHITECTURE:
  Vercel (Frontend)
    ↓ HTTPS
  Firebase Cloud Functions (sendOTP, verifyOTP)
    ├─ Rate Limiting (in-memory, 3/5 attempts)
    ├─ getSecret() fetches credentials
    └─ Access Secret Manager
        ↓
  Google Cloud Secret Manager
    ├─ TEACHER_EMAIL_USER (encrypted)
    └─ TEACHER_EMAIL_PASSWORD (encrypted)
        ↓
  Gmail SMTP (nodemailer)
    └─ Send OTP emails

DEVELOPMENT ARCHITECTURE:
  Local Machine
    ├─ .env file (TEACHER_EMAIL_USER, TEACHER_EMAIL_PASSWORD)
    └─ getSecret() falls back to .env when GCLOUD_PROJECT not set
        ↓
  Firebase Emulator Suite (local)
    └─ Test Cloud Functions locally

                       SECURITY IMPROVEMENTS
================================================================================

BEFORE:
  ❌ Credentials in .env files in repo
  ❌ No access control on credentials
  ❌ No audit trail of who accessed secrets
  ❌ Manual secret rotation (requires code change + redeploy)
  ❌ No encryption of stored credentials

AFTER:
  ✅ Credentials in Google Cloud Secret Manager (encrypted)
  ✅ Access controlled via IAM roles
  ✅ Full audit logging of all secret access
  ✅ Automatic secret rotation (no code change needed)
  ✅ Encryption at rest and in transit
  ✅ Zero-downtime secret updates

RATE LIMITING:
  ✅ sendOTP: Max 3 requests per email per 15 minutes
  ✅ verifyOTP: Max 5 attempts per email per 15 minutes
  ✅ Prevents email spam and brute-force attacks
  ✅ Returns 429 with retry information

FIRESTORE SECURITY RULES:
  ✅ Deny-by-default pattern
  ✅ Teachers can only access own classes/lists
  ✅ Students can only access own progress/responses
  ✅ Unauthenticated users blocked
  ✅ Cross-tenant data isolation enforced
